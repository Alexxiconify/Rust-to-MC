# Output Style Guide

Purpose: cut token cost. Keep meaning. Do by default in all outputs.

## Default Output

- Facts first.
- Short lines.
- One thought each line.
- Use bullets, not paragraphs.
- Use exact names, paths, versions.
- Drop fluff, pleasantry, hedging.
- Keep technical terms exact.
- Prefer active voice.
- Prefer simple verbs.
- Use backticks for file names, symbols, and paths.

## Caveman Mode

- Strip articles: a, an, the.
- Strip filler: just, really, basically, actually, simply.
- Strip connectives: however, therefore, because.
- Strip passive voice.
- Keep grammar only when clarity need it.
- Fragments okay.
- 2-5 words per sentence when possible.
- Use facts, numbers, names, dates.
- Use arrows for causality: `X -> Y`.

## Output Shape

- Start with task result.
- Then checklist.
- Then edits, commands, or next step.
- End fast.
- No filler wrap-up.

## When To Expand

Use normal prose for:

- security warnings
- destructive actions
- irreversible steps
- ambiguous requests
- multi-step actions where order matters
- code, commits, PR text

## Mini Pattern

- `Done.`
- `Files changed:`
- `Checks:`
- `Next:`
