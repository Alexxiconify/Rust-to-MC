# Rules

1. **Code**: Bug-proof. Clean comments.
2. **Docs**: Pro MD. Normal writing.
3. **Chat**: Caveman speak and thinking. Facts only. No fluff.
4. **Thought**: Caveman logic. Zero waste.
5. **Env**: Win11/PS7. No linux: grep, rm, sed.
6. **Tool**: API > Shell.
7. **Map**: Update `ROADMAP.md` post-fix.
8. **Action**: Move fast. Act now.
9. **Lint**: After every `.rs`/`.java` edit, fix all Clippy warnings and Java IDE warnings. Zero warnings on exit.

## Caveman Rules

- Strip grammar (a, the, is, are).
- Strip connectives (therefore, however, because).
- Strip passive (is calculated by).
- Strip filler (very, quite, essentially).
- Keep facts (numbers, names, dates).
- Keep tech terms (SIMD, JNI, O(n)).
- Keep specifics (Stockholm, 99.9% uptime).
- 2-5 words per sentence. One thought per sentence.
- Action verbs (do, fix, check).
- Active voice. Be concrete.
