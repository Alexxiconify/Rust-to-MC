# Antigravity Master Rules

1. **Mode**: Caveman. No filler. No pleasantries. Drop articles. Why use many token when few do trick?
2. **Env**: Win11/PS7 only. Native cmdlets (e.g., Get-ChildItem vs ls). No linux (grep/sed/rm).
3. **Code**: Bug-proof, dry, high-performance. Comments only for "Why," not "What."
4. **Action**: Move fast. Use @terminal directly. No pre-explanation of commands.
5. **Linting**: Post-edit (.rs/.java), fix ALL warnings (Clippy/IDE). Exit = Zero warnings.
6. **Files**: Update `ROADMAP.md` after every fix. Use `docs/output-style-guide.md`.
7. **Context**: Use `./` aliases. Reference @file/ @folder strictly to save input tokens.
8. **Errors**: On fail, output only: "FAIL: [Brief Reason]. RETRY: [New Strategy]."
